# User Story for Indigo's Developer Portfolio

## Primary Personas

**Persona 1: The Recruiter / Hiring Manager**
A technical recruiter or engineering manager screening candidates for junior frontend developer roles. They have 30–60 seconds to decide if a portfolio is worth a deeper look. They care about clean code, real projects, and clear communication.

**Persona 2: The Fellow Developer / Collaborator**
A peer developer, mentor, or potential collaborator who wants to see what Indigo has built, what technologies they work with, and how they think about design and code.

**Persona 3: The General Visitor**
Someone who found Indigo's portfolio through a link on a resume, LinkedIn, or GitHub. They want a quick sense of who Indigo is and what they do.

---

## User Stories

### Navigation & First Impression

- As a visitor, I want to land on a visually striking homepage with a space-themed design so that I immediately get a sense of Indigo's personality and design sensibility.
- As a visitor, I want to see a clean, intuitive navigation bar (Home, Skills, Projects, Experience, About, Contact) so I can quickly jump to the section I care about.
- As a visitor, I want smooth scrolling between sections so the single-page experience feels polished and intentional.
- As a visitor, I want the navigation to highlight the current section as I scroll so I always know where I am on the page.
- As a visitor, I want the site to load fast and look great on my phone, tablet, or desktop so I can view it on any device.

### Hero Section (Home)

- As a visitor, I want to see Indigo's name, title ("Frontend Developer"), and a brief punchy introduction so I immediately know who they are and what they do.
- As a visitor, I want to see call-to-action buttons ("View My Skills" and "Get In Touch") so I have a clear next step.
- As a visitor, I want to see animated stars, particles, and shooting stars in the background so the space theme feels immersive and memorable.
- As a recruiter, I want to see a professional presentation that balances creativity with clarity so I take the candidate seriously.

### Skills Section

- As a recruiter, I want to see a clear breakdown of Indigo's technical skills with proficiency levels so I can quickly assess if they match the role I'm hiring for.
- As a visitor, I want skills displayed as visual cards with progress bars and labels (Confident, Comfortable, Learning) so the information is easy to scan.
- As a visitor, I want to see the specific technologies listed (HTML, CSS, JavaScript, React, Tailwind CSS, TypeScript) so I know the exact tech stack.
- As a visitor, I want a color-coded legend for proficiency tiers so the skill levels are immediately understandable.

### Projects Section

- As a recruiter, I want to see real projects Indigo has built with screenshots, descriptions, and tech tags so I can evaluate their practical experience.
- As a visitor, I want projects displayed in a carousel format with the active project expanded and neighboring projects smaller on either side so each project gets its moment in the spotlight.
- As a visitor, I want forward and back arrows to navigate through all 7 projects so I can browse at my own pace.
- As a visitor, I want each project card to include: a screenshot, the project name, a brief description of what it does and what was learned, the technologies used (shown as tags), and an external link to the live project or GitHub repo.
- As a recruiter, I want to click through to see the actual deployed project or source code so I can verify the work is real.

### Experience Section

- As a recruiter, I want to see Indigo's professional timeline so I understand their career trajectory.
- As a visitor, I want the experience displayed as a vertical timeline with icons and date markers so it's visually engaging and easy to follow.
- As a visitor, I want to see Indigo's two experiences clearly: their 6-month frontend development pathway (Feb 2025 – Jul 2025) and their current Frontend Developer Internship (Feb 2026 – Present).
- As a visitor, I want each timeline entry to include the role title, organization, date range, and a brief description of what was accomplished.

### About Section

- As a visitor, I want to learn Indigo's story beyond just their resume — their journey into tech, what drives them, and what makes them unique.
- As a visitor, I want the About section to feature highlight cards (such as "6-Month Pathway," "Always Learning," "Passion for Design") so the key themes of Indigo's story are scannable.
- As a fellow developer, I want to get a sense of Indigo's learning mindset and design philosophy so I know what kind of collaborator they'd be.

### Contact Section

- As a recruiter, I want an easy way to reach out to Indigo directly from the portfolio so I don't have to search for their email separately.
- As a visitor, I want a contact form with fields for name, email, subject, and message so I can send a message without leaving the site.
- As a visitor, I want form validation that tells me if I've missed a required field or entered an invalid email so I don't accidentally submit a broken message.
- As a visitor, I want confirmation that my message was sent successfully so I know it went through.

### Downloadable Resume

- As a recruiter, I want a clearly visible button to download Indigo's resume as a PDF so I can save it for review or share it with my team.
- As a visitor, I want the resume download to work instantly without redirecting me away from the portfolio.

### Dark / Light Mode

- As a visitor, I want a toggle to switch between dark mode and light mode so I can view the site comfortably in any lighting condition.
- As a visitor, I want the site to remember my theme preference during my session so I don't have to toggle it every time I scroll.
- As a visitor, I want the space theme to adapt gracefully to light mode while still maintaining Indigo's design identity.

### Responsiveness & Accessibility

- As a mobile user, I want the navigation to collapse into a hamburger menu so it doesn't crowd the screen.
- As a visitor, I want all interactive elements (buttons, links, form fields) to be keyboard-navigable so the site is accessible.
- As a visitor, I want sufficient color contrast between text and backgrounds in both dark and light modes so I can read everything comfortably.
- As a visitor using a screen reader, I want all images to have meaningful alt text and all sections to be properly labeled so I can navigate the portfolio effectively.

### Performance & Polish

- As a visitor, I want the page to load in under 3 seconds so I don't bounce before seeing the content.
- As a visitor, I want smooth animations and transitions that enhance the experience without slowing it down.
- As a recruiter, I want a live deployed URL (on Vercel) that I can visit anytime so the portfolio is always accessible.
